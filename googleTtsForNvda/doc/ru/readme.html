Google TTS Для NVDA

Авторы: Нгуен Ан Дык, Дао Дык Чунг и Фам Хунг Выонг

Долгое время пользователи экранных дикторов мечтали использовать знакомые голоса от технологии преобразования текста в речь Google в NVDA для Windows, подобно тому, как это работает на дикторах Android.

Google TTS Для NVDA было создано, чтобы исполнить это желание. Используя технологию преобразования текста в речь Google через WebAssembly (WASM), работающую в фоновом режиме в поддерживаемой среде выполнения Chromium, такой как Google Chrome, Microsoft Edge или Brave, это дополнение привносит знакомые естественные голоса Google, улучшая вашу повседневную работу за компьютером.

После загрузки необходимых голосовых пакетов дополнение работает полностью автономно на вашем компьютере, обеспечивая быстрый отклик без необходимости постоянного подключения к интернету.

Ключевые возможности

  • Полностью автономная работа: Синтез речи выполняется локально после загрузки необходимых голосовых пакетов.
  • Разнообразие языков и голосов: Предоставляет множество голосовых пакетов Google TTS для поддерживаемых языков.
  • Быстрый отклик и низкая задержка: Использует сегментацию текста и аудиобуферизацию для повышения отзывчивости NVDA.
  • Менеджер голосов Google TTS: Просматривайте, загружайте, удаляйте и проверяйте голосовые пакеты из доступного диалога.
  • Выбор среды выполнения Chromium: Выберите Google Chrome, Microsoft Edge или Brave в качестве фоновой среды выполнения Chromium.
  • Автоматические языковые профили: Опционально используйте профили для каждого языка, чтобы разные языки могли иметь свой голос и речевые параметры.
  • Встроенное обновление дополнения: Проверяйте наличие обновлений дополнения из настроек NVDA, с опциональной автоматической проверкой при каждом запуске.

Системные требования

  • NVDA: Версия 2024.1 или новее. Дополнение поддерживает NVDA версий с 2024 по 2026 как для 32-битных (x86), так и для 64-битных (x64) сборок NVDA.
  • Браузер Chromium: На вашем компьютере должен быть установлен Google Chrome, Microsoft Edge или Brave. Если вы используете Microsoft Edge в качестве среды выполнения, также должна быть установлена среда выполнения Microsoft Edge WebView2.
  • Интерактивная пользовательская сессия Windows: Не полагайтесь на это дополнение в средах, где среда выполнения Chromium не может быть запущена, например, на экране входа в Windows, в защищенных рабочих столах, в среде предустановки Windows (Windows PE), в средах восстановления или других минимальных сессиях Windows.

Краткое руководство по началу работы

  1. Откройте диалог Выбор синтезатора NVDA, нажав NVDA+Ctrl+S, и выберите Google TTS Для NVDA.
  2. Если голосовые пакеты не установлены, NVDA отобразит сообщение о том, что голоса Google TTS Для NVDA не установлены. Нажмите ОК, чтобы открыть Менеджер голосов Google TTS и загрузить голосовой пакет. Нажмите Отмена, чтобы пока продолжить использовать ваш текущий синтезатор.
  3. В Менеджере голосов Google TTS загрузите как минимум один голосовой пакет на вкладке Загрузка.
  4. После загрузки откройте Настройки речи (NVDA+Ctrl+V), чтобы выбрать предпочитаемый язык и вариант голоса.

Управление голосами

Вы можете открыть Менеджер голосов Google TTS в любое время, чтобы добавить или удалить голосовые пакеты:

  • Нажмите сочетание клавиш NVDA+Ctrl+Shift+G или
  • Перейдите в Меню NVDA → Сервис → Менеджер голосов Google TTS...

Менеджер голосов Google TTS имеет две вкладки:

  • Установленные: Показывает голосовые пакеты, уже сохраненные на вашем компьютере. Столбец Статус сообщает, является ли каждый пакет Используемым, не поддерживается ли он встроенным движком, отсутствует ли требуемый пакет, требуется ли он другим установленным пакетом или зависит от другого пакета.
  • Загрузка: Показывает пакеты, которые можно загрузить. Столбец Статус объясняет, доступен ли пакет для загрузки, требует ли он другой пакет, установлен ли уже требующийся для него пакет или требуется ли он другим загружаемым пакетам.

На каждой вкладке элементы управления расположены в следующем порядке: Фильтр по языку, Отметить все голосовые пакеты, список пакетов, затем либо Удалить отмеченные голосовые пакеты, либо Загрузить отмеченные голосовые пакеты. Диалог также содержит строку Статус и кнопки Обновить, Открыть папку с голосовыми пакетами и Закрыть.

Чтобы добавить голоса, используйте вкладку Загрузка, отметьте нужные пакеты и нажмите Загрузить отмеченные голосовые пакеты. Чтобы удалить голоса, используйте вкладку Установленные, отметьте пакеты, которые вы больше не хотите использовать, и нажмите Удалить отмеченные голосовые пакеты.

Некоторые голосовые пакеты зависят от другого пакета. Когда вы загружаете зависимый пакет, Менеджер голосов может также загрузить требуемый пакет. Когда вы удаляете пакет, от которого зависят другие установленные пакеты, Менеджер голосов включает эти зависимые пакеты, чтобы вы не оставили неработающие голоса.

Менеджер голосов защищает вас от случайного удаления последнего используемого голосового пакета. Если Google TTS Для NVDA не является текущим синтезатором, предупреждение по умолчанию выбирает Нет. Если Google TTS Для NVDA является текущим синтезатором, Менеджер голосов просит вас сначала переключиться на другой синтезатор и оставляет последний используемый пакет установленным, если вы не переключитесь.

Вы можете использовать раскрывающийся список Фильтр по языку на любой вкладке, чтобы отобразить только голосовые пакеты для нужного языка (например, английского или вьетнамского) для более быстрого поиска голосов. Кроме того, вы можете нажать Открыть папку с голосовыми пакетами, если хотите проверить, где на вашем компьютере хранятся голосовые пакеты.

Загрузка и удаление голосов выполняются в фоновом режиме, поэтому вы можете продолжать использовать NVDA без перерывов. Объявления о ходе выполнения ограничены основными этапами и конечными результатами, чтобы не повторяться при каждом небольшом изменении процента.

Настройка

Настройки речи

Диалог настроек речи NVDA (NVDA+Ctrl+V) предоставляет параметр Паузы для Google TTS Для NVDA, независимо от того, включены ли автоматические языковые профили. Когда автоматические языковые профили отключены, диалог также предоставляет:

  • Голос: Выберите установленный язык Google TTS.
  • Вариант: Выберите имя голоса в этом языке, включая голоса Chrome OS и Google Natural, если они установлены.
  • Скорость: Настройте скорость чтения.
  • Ускорение скорости: Включите для удвоения скорости речи для высокоскоростного чтения.
  • Высота тона: Настройте высоту тона голоса.
  • Громкость: Настройте громкость речи.

Паузы: Выберите, остаются ли паузы без изменений, укорачиваются ли только в конце текста или укорачиваются по всей речи для более быстрого чтения. Этот параметр применяется ко всей речи Google TTS.

Настройки дополнения

В разделе Меню NVDA → Параметры → Настройки → Google TTS Для NVDA вы можете настроить:

  • Среда выполнения Chromium: Выберите Google Chrome, Microsoft Edge или Brave в качестве фоновой среды выполнения Chromium для дополнения. Панель настроек отображает доступность браузеров.
  • Статус среды выполнения Chromium: Просмотрите выбранную среду выполнения, доступность браузера и активную резервную среду выполнения, если она используется.
  • Использовать автоматические языковые профили: Включите автоматический выбор профиля и откройте элементы управления профилями, описанные ниже.
  • Предпочтительный язык профиля: Выберите включенный язык профиля, используемый, когда предложение неясно. Этот список может быть пустым, пока не включен хотя бы один языковой профиль.
  • Автоматический языковой профиль: Выберите, какой установленный языковой профиль редактировать.
  • Использовать этот языковой профиль: Включите выбранный языковой профиль.
  • Вариант, Скорость, Ускорение скорости, Высота тона, Громкость, Процент изменения высоты тона для заглавных букв, Произносить "заглавная" перед заглавными буквами, Сигнал для заглавных букв, Использовать функцию произнесения по буквам, если поддерживается: Настройте речевые параметры для выбранного включенного профиля.
  • Статус автоматических языковых профилей: Просмотрите, выключены ли профили, недоступны, отсутствует ли выбранный профиль или активны.
  • Обновления: Содержит элементы управления обновлением дополнения, описанные ниже.
  • Автоматически проверять наличие обновлений дополнения при запуске NVDA: Разрешить Google TTS Для NVDA выполнять проверку один раз после следующего запуска NVDA. По умолчанию выключено.
  • Статус обновления дополнения: Просмотрите установленную версию дополнения, включена ли автоматическая проверка при запуске и выполняется ли проверка в данный момент.
  • Проверить обновления...: Вручную проверить наличие последней стабильной версии дополнения.

Категорию настроек Google TTS Для NVDA также можно открыть с помощью пользовательского жеста ввода. По умолчанию для нее не назначено сочетание клавиш; назначьте его в Меню NVDA → Настройки → Жесты ввода в категории Google TTS Для NVDA.

Обновления дополнения

Откройте Меню NVDA → Параметры → Настройки → Google TTS Для NVDA и нажмите Проверить обновления..., чтобы проверить последний стабильный выпуск. При ручной проверке отображается диалог ОК, когда дополнение уже обновлено, когда невозможно подключиться к интернету или когда ответ на запрос обновления не может быть понят.

Когда доступна более новая версия, Google TTS Для NVDA показывает диалог Информация об обновлении с размером загрузки, минимальной версией NVDA, последней протестированной версией NVDA и журналом изменений. Выберите Да, чтобы загрузить пакет .nvda-addon внутри NVDA. Во время загрузки диалог Загрузка обновления Google TTS Для NVDA показывает Статус загрузки, индикатор выполнения, когда известен размер, и кнопку Отмена. Дополнение проверяет загруженный файл на соответствие размеру и контрольной сумме SHA256 из манифеста обновления, затем открывает обычный установщик дополнений NVDA.

Автоматическая проверка при запуске по умолчанию отключена. Если вы включите ее, Google TTS Для NVDA выполнит проверку один раз после следующего запуска NVDA и будет молчать, если только не доступна более новая версия. Ручная проверка остается доступной независимо от того, включена ли автоматическая проверка. Если фоновая проверка при запуске уже выполняется, пока открыта категория настроек, кнопка ручной проверки отключается до завершения этой проверки. Изменение флажка Автоматически проверять наличие обновлений дополнения при запуске NVDA влияет на будущие запуски NVDA и не прерывает уже выполняющуюся проверку.

Программа обновления также может распознавать небольшие сборки с исправлениями (hotfix) внутри одной и той же публичной версии. Это позволяет сопровождающим публиковать исправления совместимости или надежности без изменения отображаемого номера версии, в то время как NVDA все равно может предлагать более новую сборку, когда метаданные выпуска указывают на ее доступность.

Автоматические языковые профили

Когда Использовать автоматические языковые профили включено, для обнаруженных предложений используются профили дополнения для каждого языка вместо обычных настроек речи NVDA. Если включен только один языковой профиль, этот профиль используется для каждого предложения. Ваши обычные настройки языка и варианта Google TTS снова становятся доступными, когда автоматические языковые профили выключены.

Автоматические языковые профили используют встроенные библиотеки детектора CLD2 как для 32-битных (x86), так и для 64-битных (x64) сборок NVDA. Результаты CLD2 принимаются только тогда, когда они достаточно надежны для одного из включенных языков. Если текст слишком короткий или неясный, дополнение использует консервативные локальные языковые сигналы, где они доступны, затем переходит к предпочтительному включенному языку.

  1. Откройте Меню NVDA → Настройки → Параметры → Google TTS Для NVDA.
  2. Включите Использовать автоматические языковые профили.
  3. Перейдите к Предпочтительный язык профиля. В этом списке отображаются только включенные языки, поэтому он может быть пустым, пока вы не включите профиль.
  4. Выберите Автоматический языковой профиль.
  5. Установите флажок Использовать этот языковой профиль для каждого языка, для которого вы хотите использовать автоматические языковые профили. Языковые профили отключены, пока этот флажок не установлен.
  6. Для каждого включенного языка установите Вариант, Скорость, Ускорение скорости, Высота тона, Громкость, Процент изменения высоты тона для заглавных букв, Произносить "заглавная" перед заглавными буквами, Сигнал для заглавных букв и Использовать функцию произнесения по буквам, если поддерживается.
  7. Вернитесь к Предпочтительный язык профиля и выберите включенный язык, используемый, когда предложение неясно.

Скорость, высота тона и громкость используют элементы управления-ползунки, аналогичные настройкам речи NVDA. Высота тона для заглавных букв использует элемент управления "числовое поле/поле со стрелками", чтобы вы могли точно настроить его.

Общесистемные настройки NVDA, такие как уровень пунктуации и символов, автоматическое переключение диалектов, сообщение об изменении языка, нормализация Юникода, данные консорциума Юникода, включая эмодзи, дополнительные словари символов, задержанные описания символов и выбор режима речи, остаются в настройках речи NVDA.

Google TTS игнорирует собственные команды NVDA для автоматического переключения языка и диалекта для этого синтезатора, поэтому эти глобальные параметры не заставляют Google TTS менять голоса. Используйте автоматические языковые профили Google TTS, когда хотите автоматического выбора голоса/профиля в этом дополнении. Если в NVDA включена опция Сообщать об изменении языка при чтении, сообщаемые названия языков следуют за определением автоматического языкового профиля Google TTS, пока профили включены. Когда профили выключены, NVDA все еще может использовать изменения языка, предоставленные приложениями, для сообщения о языке и обработки символов/знаков, включая словари пунктуации и символов, но Google TTS не будет менять голоса по этим командам.

Автоматические языковые профили устанавливают языковой контекст перед обработкой текста NVDA, поэтому произношение символов и обработка речевых словарей NVDA остаются в обычном потоке речи для выбранного языкового контекста.

Когда автоматические языковые профили выключены, голосовые словари NVDA работают нормально для выбранного в данный момент варианта Google TTS. Когда автоматические языковые профили включены, дополнение временно использует голосовой словарь для каждого включенного языкового профиля выбранного варианта, пока NVDA обрабатывает этот сегмент. Словари NVDA по умолчанию и временные словари по-прежнему следуют обычному поведению NVDA.

Пока включены автоматические языковые профили, диалог настроек речи NVDA не будет предлагать обычные элементы управления голосом, вариантом, скоростью, ускорением скорости, высотой тона и громкостью для этого синтезатора. Вместо этого он покажет доступное для фокуса уведомление, направляющее вас в категорию настроек Google TTS Для NVDA. Паузы остаются доступными в настройках речи и применяются ко всей речи Google TTS. Обычные команды кольца настроек синтезатора, такие как NVDA+Ctrl+клавиши со стрелками на настольной клавиатуре или NVDA+Shift+Ctrl+клавиши со стрелками на клавиатуре ноутбука, объявляют это уведомление вместо изменения голоса или речевых параметров Google TTS в этом режиме. Google TTS также использует параметры заглавных букв и произнесения по буквам каждого включенного профиля, пока включены автоматические языковые профили; обычные значения настроек речи снова становятся доступными, когда автоматические языковые профили выключены. Сообщения статуса в категории настроек Google TTS Для NVDA также доступны с помощью Tab, чтобы экранные дикторы могли их объявлять.

Устранение проблем с молчанием Edge Runtime

Если вы выбираете Microsoft Edge в качестве среды выполнения Chromium, Google TTS Для NVDA также проверяет, доступна ли среда выполнения Microsoft Edge WebView2. Edge может быть установлен, когда WebView2 отсутствует или поврежден, и в этом случае дополнение не может использовать Edge для синтеза речи. Google Chrome и Brave не зависят от WebView2.

WebView2 — это среда выполнения Microsoft Edge, используемая нативными приложениями Windows для размещения веб-контента. Она связана с Microsoft Edge, но наличие обычного браузера Edge не всегда означает, что среда выполнения WebView2 установлена и работоспособна для сценариев приложений. Microsoft рекомендует Evergreen WebView2 Runtime для приложений, поскольку она является общей, автоматически обновляется, а небольшой Evergreen Bootstrapper загружает соответствующую среду выполнения для архитектуры устройства.

Когда требуется WebView2, Google TTS Для NVDA предлагает кнопки для загрузки онлайн-загрузчика Evergreen от Microsoft, открытия страницы WebView2 Microsoft для автономных установщиков или пакетов фиксированной версии или отложить изменение на потом. NVDA продолжает использовать предыдущий синтезатор, пока WebView2 не станет доступен.

Если Windows не может открыть страницу загрузки, дополнение показывает адрес загрузки в помеченном поле только для чтения и включает кнопку Копировать ссылку, чтобы вы могли вставить адрес в браузер вручную.

Для компьютера, подключенного к интернету, загрузите установщик Microsoft Edge WebView2 Evergreen Bootstrapper здесь: Загрузить среду выполнения Microsoft Edge WebView2.

Запустите установщик, перезапустите NVDA, затем снова выберите Google TTS Для NVDA. Для автономных установщиков или пакета среды выполнения WebView2 фиксированной версии откройте страницу WebView2 Microsoft: Microsoft Edge WebView2.

Примечания

  • Вам не нужно держать окно браузера Google Chrome, Microsoft Edge или Brave открытым во время использования NVDA. Дополнение автоматически подключается и обрабатывает речь в фоновом режиме.
  • Когда NVDA впервые начинает говорить после запуска, голосам может потребоваться короткое время для инициализации.
